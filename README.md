本プロジェクトは、Kaggleのコンペティション「House Prices - Advanced Regression Techniques」に対する機械学習アプローチの記録です。
アイオワ州エイムズの住宅データセット（79の説明変数）を使用し、住宅価格（SalePrice）を予測する回帰モデルを構築しました。

## 1. 目的変数の変換
住宅価格（SalePrice）は右裾が長い（富裕層向けの極端に高価な物件が存在する）対数正規分布に近い形状をしていました。
回帰モデルが極端な外れ値に引っ張られるのを防ぎ、予測精度を安定させるため、目的変数に対して 対数変換（np.log1p） を適用し、正規分布に近似させてから学習を行いました。

## 2. 根拠に基づく欠損値の処理 
本データセットの最大の特徴は、「欠損値（NA）が『データの記録漏れ』ではなく、『その設備が存在しない』ことを意味する」点です。全体を一律の手法で埋めるのではなく、ドキュメントと可視化に基づく4つのアプローチを採用しました。

* カテゴリ変数の「設備なし」: PoolQC（99%欠損）や Alley、Fence などは、欠損自体が「プールなし」「フェンスなし」という重要な情報であるため、新たなカテゴリ 'None' として補完。
* 数値変数の「設備なし」: ガレージが存在しない場合の GarageArea や GarageYrBlt は、論理的に 0 で補完。
* 地域性に依存する欠損値: LotFrontage（道路に面した間口の長さ、約17%欠損）については、箱ひげ図を用いて「間口の広さは地域（Neighborhood）ごとに差がある」ことを視覚的に証明した上で、各物件が属する Neighborhood の中央値で補完。
* 完全な記録漏れ: Electrical など、ごくわずかな欠損は最頻値で補完。

## 3. 特徴量エンジニアリング
人間の「家を買う時の直感」をモデルに反映させるため、仮説に基づき新しい特徴量を作成し、統計的なテストでその有効性を検証しました。

### 総面積（TotalSF）の合成
* 仮説: 買い手は「1階の面積」や「地下室の面積」を個別に見るよりも、家全体の「総面積」を重視するのではないか？
* 処理: TotalBsmtSF + 1stFlrSF + 2ndFlrSF を合算し、TotalSF を作成。
* 根拠: 既存の地上面積（GrLivArea）と価格の相関係数（r = 0.7009）に対し、作成した TotalSF は r = 0.7773 へと大幅に向上。p値も極めて小さく、より強力な予測因子であることを統計的・視覚的（散布図と回帰直線）に証明した上で採用しました。

### 順序尺度の数値化
* KitchenQual（キッチンの品質）などの評価データ（Ex, Gd, TA, Fa, Po）に対し、箱ひげ図を作成。
* 評価が下がるにつれて価格帯が綺麗に下降する（階段状になる）傾向を確認した上で、「Ex=5, Gd=4... None=0」と大小関係を持たせた連続値に変換しました。

## 4. データリークの防止とエンコーディング
* One-Hot Encodingと列の同期: 名義尺度（順序のないカテゴリ）はダミー変数化。その際、学習データとテストデータで未知のカテゴリによる列数の不一致を防ぐため、align(join='inner') を用いて特徴量の形状を完全に一致（200列以上）させました。
* Pipelineの構築: 交差検証（Cross-Validation）時に検証データの情報が学習データに漏れる（データリーク）のを完全に防ぐため、StandardScaler とモデルを Pipeline で結合し、正しい手順でスケーリングと評価を行いました。

## 5. モデル構築と評価 
多重共線性と過学習を防ぐため、まずは線形モデル群でベースラインを構築し、その後非線形モデルで精度を引き上げました。評価指標は Kaggle 指定の RMSE（Root Mean Squared Error）です。

### Baseline: 線形モデルの比較 (Ridge vs Lasso)
* Linear Regression: RMSE 0.21190（多重共線性により精度悪化）
* Ridge Regression (alpha=300.0): RMSE 0.14778
* 考察: 強めのペナルティをかけた Ridge回帰が線形モデルでトップ。特徴量重要度（係数）を確認すると、自作した TotalSF が価格を押し上げるトップ要因にランクインしており、特徴量設計の成功が裏付けられました。

### Champion Model: チューニングされた LightGBM
線形モデルでは捉えきれない複雑な関係性（Aの条件かつBの条件の時のみ価格が上がる等）を学習させるため、決定木ベースの LightGBM を採用し、GridSearchCV でハイパーパラメータを最適化しました。

* Best Parameters: learning_rate=0.05, max_depth=2, n_estimators=700, num_leaves=15
* Final CV RMSE: 0.12964
* 考察: 200列以上の特徴量に対して過学習を起こさないよう、「極めて浅い木（max_depth=2）を大量に作る（n_estimators=700）」という戦略をモデルが自動的に選択し、Ridge回帰の壁を大きく打ち破るスコアを記録しました。
